pmCrypto · 这个项目到底在做什么

一份把代码里的真实机制讲清楚的全景说明 —— 不美化、不含糊、每条都有依据

§一句话概括

这是一个盯着 Polymarket（预测市场） 的自动交易系统。它做两件事： ① 找市场里机械的算术错价（无风险套利）； ② 对加密货币价格类市场，用数学模型算出"真实概率"，找出被市场定错价的机会。 默认只提示、不下单，要真金白银交易需手动逐层解锁。

Q你最关心的几个问题

先直接回答你的疑问，纠正几个常见误解。

是"5 分钟 / 15 分钟 BTC 涨跌预测"吗？ 不是
这是最大的误解。系统预测的不是"未来 5 分钟 BTC 涨还是跌"。它处理的是这种形式的市场：
"到某个到期日，BTC 是否高于/低于某个价格？"——例如 "Will Bitcoin be above $60,000 on 2025-12-31?"。 它解析出三样东西：币种（BTC）、目标价（$60,000）、方向（above/below）， 完全没有"5分钟/15分钟"这种时间窗口维度。到期时间从几小时到几个月不等，题目是固定的。
多久预测/扫描一次？
每 45 秒扫描一轮（可调，poll_interval_sec）。但请注意区分两件事： "多久扫一眼" = 45 秒；"市场多久出结果" = 由每个市场自己的到期日决定（各不相同）。 你扫得再勤，一个"本周五结算"的市场，一周也只产生 1 个最终结果。
支持哪些币？
加密定价模型支持 BTC、ETH、SOL、DOGE 四种。其它币种的市场，模型不参与。
能预测世界杯 / 体育 / 政治这些热门市场吗？ 部分/需分清
加密模型完全不碰它们——遇到非加密市场，解析直接返回"无法识别"，跳过，绝不猜。 所以系统不会用价格模型给世界杯"预测下注"。
唯一例外：无风险套利策略与主题无关，任何市场（含世界杯）只要出现"买 Yes + 买 No 的价格加起来 < 1" 这种机械错价，它都会提示。但那是捡算术便宜，不是预测比赛结果。
它现在会自动花我的钱吗？ 默认不会
默认是"只提示、不下单"模式（干跑）。要真金白银交易，必须逐层通过：edge 验证达标 + 钱包凭证齐全 + 手动点"启用真钱"。任一不满足就绝不花钱。详见下方安全闸门。

1到底交易哪些市场，用几种策略

平台是 Polymarket。市场以二元 Yes/No 为主，也有多结果市场。系统有三种信号来源：

策略 A · 无风险套利（与主题无关）最稳

不预测任何东西，只找机械的算术错价：

  • Yes/No 互补偏差：如果"买 Yes 的价 + 买 No 的价 < 1"，同时买两边，到期必有一边赢，锁定无风险差价。
  • 多结果概率和偏差：多个互斥结果的买入价加起来 < 1，全买锁利。

这类机会稀少但确定性高，对世界杯/政治/加密任何市场都成立。（注：多腿套利需原子成交，当前版本只提示、不自动执行。）

策略 B · 方向性统计信号

综合动量（价格趋势）、盘口失衡（买卖盘厚薄）、量价背离三个统计信号，估一个概率，和市场报价比找偏差。 这是纯统计、无基本面模型，对陌生市场容易误导——所以必须经验证才允许真下单。

策略 C · 加密价格误定价（最有效，下一节详解）核心

对 BTC/ETH/SOL/DOGE 的"是否突破某价"市场，用数学模型算真实概率找错价。这是系统的主打。

2加密预测的原理（核心机制）

这是系统最有含金量的部分，值得看懂。它其实是一套"期权定价"的思路。

要回答的问题

市场问："到期日 T，BTC 会不会高于 $K？"并给了一个报价 q（比如 0.35，即市场认为 35% 概率）。 系统要独立算出自己认为的真实概率 p，如果 p 明显 > q，说明市场把它定便宜了 → 值得买。

怎么算这个真实概率 p

用对数正态模型（几何布朗运动，Black-Scholes 同款假设）——金融界给期权定价的标准工具：

P(到期时 ST > K) = Φ(d₂)
d₂ = [ ln(S / K) − ½·σ²·T ] / (σ·√T)

S = 当前现价   K = 目标价   σ = 年化波动率   T = 距到期时间(年)   Φ = 标准正态累积分布

直觉解释：现价离目标价越近、时间越长、波动越大 → 突破的概率越高。这三个因素被数学严格地组合起来。

三个输入从哪来

现价 S实时从 Coinbase / Binance 拉取 到期时间 T从市场自带的到期日算出 波动率 σ用近 30 天真实 K 线算出的"已实现波动率"（对数收益标准差×√365），拉不到才降级到常数
关于波动率 σ 的重要改进：早期版本 σ 是写死的常数（BTC 55%）。实测发现 BTC 近期真实波动率约 38%——常数高估了 44%！这会系统性地高估"看涨概率"、制造假机会。现已改为用真实历史 K 线校准， 从根上消除了这个偏差。

置信度

模型对自己的信心随到期时间变化：越临近到期越可靠（>30天→0.3 低信心；≤1天→0.95 高信心）。 因为越临近到期，波动率假设的影响越小，极端情况下退化为"现价是否已经过线"，这时几乎不会错。

3它科学吗？—— 诚实的局限

模型是严谨的，但有真实的边界。这些是代码作者自己在注释里承认的，不是我加的。

① 波动率是"已实现"不是"隐含"。我们用"过去 30 天实际波动"，而市场定价参考的是"未来预期波动"（隐含波动率）。 两者通常接近，但重大事件前会背离。这比写死常数好得多，但仍非完美。
② 滑点是"估"的。真实下单时盘口在变、还有别的机器人抢单。系统刻意把滑点估得偏保守（放大 1.5 倍）， 宁可少算机会也不高估收益——因为高估收益 = 拿真钱去亏。
③ edge 是估计，必须回测验证。模型算出的"真实概率"终究是估计。所以系统要求：只有用真实已结算结果 验证过、且统计上显著为正（95% 置信区间下界 > 0、样本 ≥ 30 个已结算市场），才允许上真钱。 样本不足时会诚实地判"证据不足，别上真钱"。

一句话：方法是科学的（标准期权定价 + 真实波动率 + 统计显著性检验），但它给的是"概率优势"不是"确定性"。 能不能赚钱，最终要靠真实结算数据来验证，而不是模型自己说了算。

4怎么决定下多少钱

用 Kelly 公式做"复利最优下注"，但层层设限，绝不梭哈。

Kelly 最优比例 f* = edge / (1 − q) = (p − q) / (1 − q)

然后经过三重削减取最小值，确保永不破产：

  • 自适应 Kelly 分数：只用 ¼~½ Kelly（信心越高越接近 ½），比满 Kelly 保守得多。
  • 单笔硬封顶：最多占余额 20%。
  • 总仓位上限 + 流动性限制：所有在场仓位合计最多 50% 余额，且不超过市场能吃下的深度。
复利核心：每次下单前用真实链上余额刷新基数。赢了余额变大 → 下一注自动变大；输了自动变小。 这是"自动复利"，但前提是"永不破产"——所以上面的削减和下面的熔断一个都不能少。

5防止乱下单 / 亏损的闸门

从"想下单"到"真花钱"，要连过 6 道门；运行中还有 4 道自动熔断。

下单前的 6 道门（任一不过就拒绝）

1全局急停门 —— 系统处于急停态，一律不下单。
2干跑门（默认开）—— 走完全部逻辑但只打印、不发单。默认就是这个，不显式关掉不会花钱。
3edge 验证门 —— 未经真实结算验证（edge_verified）不许真下单。
4凭证门 —— .env 里钱包私钥 + API 凭证不齐，拒绝。
5机会类型门 —— 目前只自动执行单腿方向性单；多腿套利需原子成交，只提示。
6在场持仓数门 —— 钱包在场持仓达上限（默认 10 笔）拒绝开新仓，平仓后自动腾额度。

运行中的 4 道自动熔断

  • API 连续失败 10 次 → 急停（怕数据断流下错单）。
  • 当日亏损 触及上限 → 急停。
  • 连续亏损 达 5 笔 → 急停（信号可能失效）。
  • 余额跌破绝对底线 → 彻底停手。
下单方式用 FOK 市价单（要么立即全部成交、要么取消），不会留下"挂单未成交却记账"的幽灵持仓； 成交后写台账，市场结算时自动回填盈亏、驱动上面的熔断。

6到底怎么才能赚钱

赚钱不靠"预测得多"，靠"只在验证过的优势上，用复利下注"。顺序不能颠倒。

1空跑攒样本：让系统持续记录信号，等市场陆续到期结算。优先扫"近到期"市场（可调）能加速这一步。
2跑 edge 验证：攒够 ≥30 个已结算市场后，看历史信号的真实收益。95% 置信区间下界 > 0 才算"edge 是真的"。
3微额灰度实盘：验证通过后，把单笔压到 0.5~1 USDC 开真钱，每日对账"预期盈亏 vs 实际盈亏"。这一步戳穿模型偏差。
4Kelly 复利放大：只有实盘对账证明"预期≈实际"了，才逐步放大仓位，让复利发挥作用。
最诚实的一句话：快速积累的是"意见"（每 45 秒一堆信号），不是"被验证的意见"。 赚钱的前提是后者。任何"高频 = 快速稳赚"的直觉，在预测市场里几乎都通向亏损。 尊重"攒样本 → 验证 → 微额实盘 → 才放大"这个顺序，才是这套策略能稳定复利的真正前提。
